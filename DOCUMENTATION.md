# 古字通创作说明文档

## 一、作品概述

### 1.1 项目背景

古字通是一款面向初中生的文言文学习微信小程序。随着部编版教材的使用，文言文在中考中的占比逐年提升，但初中生在学习文言文时普遍存在以下困难：

- 词汇量匮乏，120个实词记忆困难
- 难以理解古今异义词的特殊用法
- 缺乏语感，无法准确翻译句子
- 课后复习巩固缺乏有效方法

### 1. 项目定位

古字通定位为**初中生文言文学习工具**，旨在通过数字化手段解决传统文言文学习中的痛点，让学习更加高效、有趣。

### 1.3 核心目标

1. **查词** - 快速查询文言字词含义和例句
2. **学习** - 掌握150个初中必学实词
3. **翻译** - 辅助翻译简短文言文段落
4. **复习** - 基于艾宾浩斯遗忘曲线的科学复习

---

## 二、拟解决的关键问题

### 2.1 词汇记忆困难

**问题**：初中阶段需要掌握120个实词，学生往往死记硬背，容易遗忘。

**解决方案**：
- 150个实词分组学习，每组3/5/10词可选
- 多种练习模式（句子选择、多选题）加强记忆
- 艾宾浩斯遗忘曲线复习，间隔1/2/4/7/15/30天

### 2.2 缺乏语感

**问题**：学生只知其意，不知其用，无法准确理解句子含义。

**解决方案**：
- 每个词配备出自教材的经典例句
- 句子选择题型，让学生在语境中理解词义
- 翻译功能辅助理解完整句子

### 2.3 复习效率低

**问题**：课后复习无计划，学到后面忘前面。

**解决方案**：
- 记录每次学习进度
- 智能计算复习时间
- 复习提醒机制

### 2.4 学习数据不同步

**问题**：换设备后学习记录丢失。

**解决方案**：
- 云端数据同步（可选）
- 服务器API数据备份

## 三、功能说明

### 3.1 查词功能

**入口**：首页查词输入框

**流程**：
1. 用户输入字词
2. 后端检索本地词库
3. RAG检索古诗文库补充
4. 返回释义、例句、出处

**特点**：
- 支持模糊查询
- 收藏功能
- 查询历史记录

### 3.2 学习功能

**入口**：底部导航栏"学习"

**流程**：
1. 查看学习进度卡片
2. 开始学习/温故复习
3. 展示环节：浏览词义
4. 练习环节：答题巩固
5. 本组完成，循环下一组

**练习模式**：
- **句子选择**：从句子中选出错词对应的意思
- **多选题**：选择正确释义（可能有多个）

**特点**：
- 艾宾浩斯遗忘曲线
- 组式学习，可自定义每组词数
- 进度实时保存

### 3.3 翻译功能

**入口**：底部导航栏"翻译"

**流程**：
1. 用户输入文言文
2. WebSocket流式调用智谱GLM API
3. 流式返回翻译结果

**特点**：
- 流式输出，体验好
- 支持上下文对话
- 翻译历史记录

### 3.4 朗读功能

**入口**：学习页面入口 或 底部导航"朗读"

**内容**：150个古字标准发音

**流程**：
1. 选择要朗读的字
2. TTS语音播放
3. 跟读练习

**特点**：
- 解决方言区发音问题
- 150字正音

### 3.5 个人中心

**入口**：底部导航栏"我的"

**功能**：
- 学习统计（已学词数、连续学习天数）
- 学习进度饼图（未学习/复习中/复习完成）
- 收藏夹
- 学习记录
- 云端同步
- 学习设置（每组词数）

## 六、应用反思

### 6.1 做得好的地方

1. **艾宾浩斯复习**：科学有效的复习机制，学生反馈良好
2. **分组学习**：降低学习压力，提高完成率
3. **多种练习模式**：增加学习趣味性
4. **数据同步**：云端备份，学习记录不丢失

### 6.2 需要改进的地方

1. **词库内容**：可增加虚词、实词用法辨析
2. **练习题型**：可增加听写、填空等模式
3. **社交功能**：可增加学习排行榜、成就系统
4. **AI辅助**：可增加智能出题、错题分析

### 6.3 未来规划

1. 增加词性、用法详解
2. 增加中考真题练习
3. 增加语法讲解模块
4. 支持小程序分享到微信群
5. 开发PC端网页版

### 6.4 经验总结

1. **小步快跑**：先完成基础功能，再迭代优化
2. **用户反馈**：及时收集用户意见，快速改进
3. **技术选型**：根据实际需求选择技术栈，不过度设计
4. **数据驱动**：重视数据存储和同步，提升用户体验
